# API Reference

The Backend exposes a complete RESTful API that is simultaneously documented via Swagger and ReDoc, supporting a bilingual format (English and Portuguese).

## Main Endpoints

### Authentication (`/api/Auth`)
- `POST /login`: Accepts credentials and returns secure and strict HttpOnly Cookies containing authorization JWTs. High rigorous rate-limiting protection is enforced.
- `POST /register`: Registers an account by following rules handled inside `strongPasswordValidator`.

### Admin Panel (`/api/AdminPanel`)
Restricted endpoints specifically for Administrators (verified in `RouteProtectionMiddleware`).
- `GET /logs`: Reclaims detailed audit-centric logs generated by the Audit Service.
- `GET /contracts`: Retains complete customer interactions alongside tracking system history.

### Payments and Agreements (`/api/Agreements`)
- `POST /formalize`: Flagship trigger endpoint to spawn a billet or launch PIX payments. It encapsulates a protection standard known as *Distributed Locking*, backed by Redis.
- `GET /billet/{id}`: The gateway that hands back IFRAME sources and proxy headers to visualize billets minus any flawed Base64 leakages.

## Rate Limiting & Payload Restrictions
The application ensures horizontal scalability by throttling vast payloads. Any request towering above the preset capacity bound (normally a few megabytes) or generating massive throughput yields an immediate `429 Too Many Requests` status block.
